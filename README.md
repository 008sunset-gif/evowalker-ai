# EvoWalker AI 🚶🧬

**EvoWalker AI** は、AI（自律型ロボット個体群）が「二足歩行」を自律的に学習し、世代交代を繰り返して「直進歩行技術」を磨き上げていく過程を、3Dビジュアルと数理的なデータで観察・分析できる**AI歩行進化シミュレーター（Webアプリケーション）**です。

就職活動における「Web開発実績」「プログラミング能力」「UI/UX・デザイン設計経験」をアピールするためのポートフォリオ成果物として開発されました。

---

## 🚀 プロジェクト概要

「遺伝的アルゴリズムによる学習プロセス」を直感的に視覚化し、非技術者からエンジニアまで誰でも楽しめるように設計された、インストール不要のフロントエンド完結アプリです。
AIの歩行方針（安定重視、バランス、スピード重視）、学習スタイル（変異の強さ）、同時歩行個体数、世代数をカスタマイズし、世代を追うごとにロボットたちがふらつきや転倒を克服し、まっすぐ安定して歩くようになる進化プロセスを観察・診断できます。

---

## 💡 解決したい課題 ＆ 就活でのアピールポイント

1. **AI学習プロセスの直感的可視化**:
   遺伝的アルゴリズムは結果だけを見ても、裏側の学習プロセス（なぜその挙動になるのか）が分かりにくいものです。本アプリは、世代交代ごとの「スコアの向上」「安定歩行率の上昇」「転倒率の低下」をリアルタイムの3D描画とSVGグラフで明示し、AIの「試行錯誤と進化」を誰もが体感できる教育デモに昇華させました。
2. **React Three Fiber によるリアルタイム3D描画実装力**:
   **React Three Fiber（Three.js）** を用いて、ブラウザ上で歩行ロボット個体群・レーン環境・パーティクル演出をリアルタイムに3D描画しています。歩行の物理挙動（サイン波ベースの歩容と重心バランス）は外部物理エンジンに頼らず**自作の簡易歩行物理**として実装しており、Reactの状態管理と3D描画ループ（`useFrame`）を破綻なく同期させる設計力をアピールしています。
3. **クリーンなUI/UX設計 (デザインアピール)**:
   単に「動く技術デモ」に留まらず、余白、視線誘導、ボタン優先度を設計したUIを採用。シミュレーション中の「待機ストレス」を軽減する「早期終了条件」や「途中分析導線」、倍速再生機能を実装し、製品レベルのUX完成度を追求しています。

## 🌟 主な機能と実装工夫

### 1. リアルタイム3D歩行シミュレーター (React Three Fiber / Three.js)

* `@react-three/fiber` と `@react-three/drei` を用い、歩行ロボット・直線/細道/障害物レーン・追従カメラ・パーティクル（足元の波紋、転倒時の火花、ゴール時の紙吹雪）を3D空間に描画。
* ロボットは頭部・胴体・アーム・脚が動的に動くデザインを適用し、進行速度やバランスの崩れに合わせて体の傾き（ふらつき）が動的に変化します。
* 物理状態（前進方向 `x`、横ずれ `y`、重心の傾き `wobbleAngle`）と3D描画座標を一貫した変換ルール（`worldZ = -y`）で結び付け、当たり判定と見た目を一致させています。
* ゴール到達時には紙吹雪の演出、転倒時には火花の演出を行い、成否を視覚的に明確化しています。

### 2. 遺伝的アルゴリズム（Genetic Algorithm）

* 各個体の歩行特性を、以下の12個の数理パラメータからなる「ゲノム（DNA）」として定義しています。
  `strideLength`（歩幅）, `stepRhythm`（歩行リズム）, `balanceCorrection`（姿勢補正力）, `lateralDrift`（横流れの癖）, `wobbleStrength`（ふらつき量）, `speedFactor`（前進速度ベース）, `steeringBias`（左右の偏り）, `recoveryAbility`（立て直す力）, `lateralAmplitude`（蛇行の大きさ）, `lateralFrequency`（蛇行の周期）, `lateralPhase`（蛇行の初期位相）, `centerPull`（中央へ戻る力）。
* 障害物コースは、障害物を「検知して避ける」専用ロジックではなく、上記の歩行パターン遺伝子の違いによって各個体の軌道が変わり、結果的に避けられた個体が選抜される設計です。
* **適合度評価（Fitness Score）**: 前進距離、生存フレーム時間、レーン中央維持ボーナス、姿勢安定ボーナスを加算し、転倒・レーン外・立ち往生にはペナルティを、ゴール到達にはボーナスを与えるスコアシステム。
* **エリート選抜 (Elitism)**: 毎世代の上位20%をエリートとして選別し、最高成績を残したベスト個体のゲノムは無変更で次世代へ完全コピー（エリート保護）。
* **突然変異 (Mutation)**: 残りの個体は選抜されたエリートを親とし、学習スタイル（変異の強さ）に応じた変異率（`5% / 15% / 30%`）で正規分布風のランダムノイズをゲノムに加えて次世代を生成します。
  ※本デモでは挙動の説明性と実装の安定性を優先し、交叉（Crossover）ではなく「上位個体の保護＋突然変異＋少数のランダム新規個体」によるシンプルなGA構成を採用しています（ニューラルネットワークも不使用）。

### 3. 進化統計データのSVG可視化

* 「平均前進距離」「ゴール率」「安定歩行率」「転倒率」「コースアウト率」の推移を、自作のSVGグラフで動的に描画。各グラフには指標の意味が直感的にわかる説明文を配備。
* 「第1世代 ➔ 最終世代」での数値の変化を一目で対比できる「進化比較テーブル」と、最終世代の成績上位3ロボットの詳細なスタッツ（ID、スコア、距離、生存時間、状態）を網羅したランキングリストをサイドに表示。

### 4. AI歩行診断レポート（ルールベース評価）

* 蓄積されたすべての世代結果を、ルールベース評価エンジンが解析。
* スコア改善率と転倒率の変化から、最終歩行モデルを4つのタイプ（`理想的進化型`、`安定学習・リスク回避型`、`攻撃的進化・ハイリスク型`、`学習停滞型`）に分類し、総合レーティング（S〜C）を算出。
* 「改善された点」「まだ残る弱点」「次回おすすめ設定」を結論ファーストで優先表示し、長文の詳細レポートはアコーディオン内に折りたたんで情報過多を防ぎました。

## 🎨 画面遷移 ＆ 設計意図

1.  **トップ画面 (`/`)**: アプリのイントロダクションとスタートCTA。歩行ロボットの3Dプレビューを添えた、就活で見せる作品の「表紙」としてのヒーローセクション。
2.  **レーン選択画面 (`/scenario`)**: 直線レーン（広い/初級）、細道レーン（狭い/中級）、障害物レーン（立体ブロック障害物あり/上級）の3環境。（内部的な scenarioId は順に `city` / `highway` / `circuit` で、表示上は 直線／細道／障害物 レーンとして扱っています。）
3.  **AI設定画面 (`/config`)**: AIの歩行方針（性格）、学習スタイル（変異の強さ）、同時歩行個体数、最大学習世代数をカスタマイズ。
4.  **ブリーフィング画面 (`/briefing`)**: 選択された条件と、それによってAIの初期パラメータにどんなバイアスが反映されたかを説明。
5.  **シミュレーション画面 (`/simulation`)**: React Three Fiber による3D歩行シミュレーション。世代交代後のフリーズを防ぐ「React-3D状態同期ロック（Ref同期）」を搭載し、倍速（1倍速/2倍速/5倍速）切り替え、生存個体数・安定歩行率・最高距離のリアルタイムテレメトリ、スクロールログを完備。
6.  **世代分析画面 (`/analysis`)**: 各種統計指標のSVGグラフ、初期/最終世代の比較対照表、上位個体ランキング。
7.  **最終診断画面 (`/result`)**: S〜Cの評価ランク、改善された点・弱点・おすすめ設定、および開発アーキテクチャ（遺伝的アルゴリズムの仕組み等）を語るアコーディオン詳細レポート。

## 🛠️ 技術構成 (Technology Stack)

*   **フロントエンド**: React 19.x, React Router v7.x
*   **3D描画**: React Three Fiber (`@react-three/fiber`), drei (`@react-three/drei`), Three.js
*   **物理・進化ロジック**: 自作の簡易歩行物理（サイン波ベースの歩容と重心バランス）＋ 遺伝的アルゴリズム（エリート選抜＋突然変異）を Pure TypeScript で実装
*   **ビルドツール / 言語**: Vite 8.x, TypeScript (Strict Mode)
*   **スタイリング**: Vanilla CSS (CSS変数を用いたテーマ設計, Google Fonts Outfit & Inter)
*   **ビルド・型チェック**: `npx tsc --noEmit -p tsconfig.app.json` & `npm run build` がエラーなくコンパイルされることを検証済み。

## ⚡ ローカル起動方法

事前に [Node.js](https://nodejs.org/) がインストールされていることを確認してください。

1.  **リポジトリのクローン**
    ```bash
    git clone https://github.com/your-username/evowalker-ai.git
    cd evowalker-ai
    ```
2.  **依存関係のインストール**
    ```bash
    npm install
    ```
3.  **開発サーバーの起動**
    ```bash
    npm run dev
    ```
    起動後、ブラウザで `http://localhost:5173/` を開きます。
4.  **本番ビルド**
    ```bash
    npm run build
    ```
    `dist/` ディレクトリに最適化された静的ファイルが出力されます。

## 📷 UIデザイン & 設計アピール

*※就職活動の面接時やポートフォリオでアピールするための、主要画面のUI設計意図です。*

*   **配色設計**: 進行・主要アクションをブルー、成功・ゴールをグリーン、転倒・リタイア・警告をレッドで機能的に配置し、状態が直感的に伝わるようにしています。
*   **視線誘導と導線**: 重要なボタン（Primary CTA）には強調スタイルを適用し、次に取るべきアクションでユーザーが迷わない動線を引いています。シミュレーションでは倍速再生・途中分析導線・早期終了でストレスを軽減しています。
