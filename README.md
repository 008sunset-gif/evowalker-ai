# EvoWalker AI

EvoWalker AI は、AI（自律型ロボットの個体群）が二足歩行を学習し、世代交代を繰り返して直進歩行を上達させていく過程を、3D描画と統計データで観察できる Web アプリケーションです。

Web 開発・リアルタイム 3D 描画・遺伝的アルゴリズムの実装を試すために作った個人開発作品で、企画・設計・実装からテスト・ビルド検証まで単独で担当しています。

---

## 概要

遺伝的アルゴリズムによる学習の過程をブラウザ上で可視化する、インストール不要のフロントエンド完結アプリです。AI の歩行方針（安定重視／バランス／スピード重視）、学習スタイル（変異の強さ）、同時歩行する個体数、世代数を設定すると、世代を追うごとにロボットがふらつきや転倒を減らし、まっすぐ歩くようになっていく様子を観察できます。

物理計算と進化ロジックはすべてクライアント側で動作し、外部 API やサーバーは使いません。

---

## 設計上の工夫

1. **学習プロセスの可視化**
   遺伝的アルゴリズムは結果だけを見ても途中経過が分かりにくいため、世代ごとのスコア・安定歩行率・転倒率の変化を 3D 描画と数値で並べて見られるようにしました。

2. **React Three Fiber によるリアルタイム 3D 描画**
   React Three Fiber（Three.js）で、歩行ロボットの個体群・レーン環境・パーティクル演出をブラウザ上に描画しています。歩行の挙動は外部物理エンジンを使わず、サイン波ベースの歩容と重心バランスを組み合わせた簡易物理として自前で実装し、React の状態管理と描画ループ（`useFrame`）を同期させています。

3. **UI/UX**
   シミュレーション中の待ち時間を減らすため、早期終了条件・途中で分析へ進める導線・倍速再生を用意しています。状態（進行・成功・失敗）が色で伝わるよう配色を整理しました。

## 主な機能

### 1. リアルタイム 3D 歩行シミュレーター（React Three Fiber / Three.js）

* `@react-three/fiber` と `@react-three/drei` を用い、歩行ロボット・直線／細道／障害物レーン・追従カメラ・パーティクル（足元の波紋、転倒時の火花、ゴール時の紙吹雪）を 3D 空間に描画します。
* ロボットは頭部・胴体・腕・脚が動き、進行速度やバランスの崩れに合わせて体の傾き（ふらつき）が変化します。
* 物理状態（前進方向 `x`、横ずれ `y`、重心の傾き `wobbleAngle`）と 3D 描画座標を一定の変換ルール（`worldZ = -y`）で対応させ、当たり判定と見た目を一致させています。

### 2. 遺伝的アルゴリズム

* 各個体の歩行特性を、次の 13 個の数値パラメータからなるゲノムとして定義しています。
  `strideLength`（歩幅）, `stepRhythm`（歩行リズム）, `balanceCorrection`（姿勢補正力）, `lateralDrift`（横流れの癖）, `wobbleStrength`（ふらつき量）, `speedFactor`（前進速度のベース）, `steeringBias`（左右の偏り）, `recoveryAbility`（立て直す力）, `lateralAmplitude`（蛇行の大きさ）, `lateralFrequency`（蛇行の周期）, `lateralPhase`（蛇行の初期位相）, `centerPull`（中央へ戻る力）, `obstacleAvoidance`（障害物を避ける強さ）。
* **障害物レーンの回避学習**: 障害物レーン（circuit）では、各個体が前方センサーで一定距離（`LOOKAHEAD`）以内の障害物を検知し、レーン内で現在地に最も近い空き帯へ横移動します。回避の強さは遺伝子 `obstacleAvoidance` に比例します。障害物に当たると即失敗になるため、世代交代を通じて「避けながら前進できる個体」が選抜され、回避行動が上達していきます。直線・細道レーンでは回避は働きません。
* **適合度評価（Fitness Score）**: 前進距離、生存フレーム時間、レーン中央の維持、姿勢の安定にスコアを加算し、転倒・レーン外・立ち往生・障害物衝突にはペナルティ、ゴール到達にはボーナスを与えます。障害物レーンでは通過した障害物の数もボーナスに加えます。
* **エリート選抜（Elitism）**: 各世代の上位（個体数の約 20%、3〜6 体）をエリートとして残し、前世代のベスト個体は無変更で次世代へコピーします。
* **突然変異（Mutation）**: 残りの個体はエリートを親とし、学習スタイルに応じた変異率（`5% / 15% / 30%`）でゲノムにランダムなノイズを加えて生成します。さらに末尾の少数体は完全にランダムな新規個体に置き換え、多様性を保ちます。
  交叉（Crossover）やニューラルネットワークは使わず、エリート保護＋突然変異＋少数のランダム新規個体というシンプルな構成にしています。

### 3. 進化統計の表示

* 「平均前進距離」「ゴール率」「安定歩行率」「転倒率」「コースアウト率」について、第 1 世代と最終世代を Before/After で比較するカードを表示します。バーが最終世代の値、細い縦線が第 1 世代の位置を表し、改善は緑・悪化は赤で示します。
* 最終世代の成績上位ロボットのスタッツ（ID、スコア、距離、生存時間、状態）をランキングとして並べます。

### 4. 歩行診断レポート（ルールベース）

* 全世代の結果をルールベースの評価ロジックで解析します。
* スコアの改善率と転倒率の変化から、最終モデルを 4 タイプ（理想的進化型／安定学習・リスク回避型／攻撃的進化・ハイリスク型／学習停滞型）に分類し、S〜C のレーティングを算出します。
* 「改善された点」「残る弱点」「次回おすすめ設定」を先に表示し、詳細はアコーディオン内に折りたたんでいます。

## 画面構成

1. **トップ画面（`/`）**: アプリの紹介とスタート導線。歩行ロボットの 3D プレビューを表示します。
2. **レーン選択画面（`/scenario`）**: 直線レーン（広い／初級）、細道レーン（狭い／中級）、障害物レーン（ブロック障害物あり／上級）の 3 環境から選びます。内部の scenarioId は順に `city` / `highway` / `circuit` で、表示上は直線／細道／障害物レーンとして扱っています。
3. **AI 設定画面（`/config`）**: 歩行方針、学習スタイル（変異の強さ）、同時歩行個体数、最大学習世代数を設定します。
4. **シミュレーション画面（`/simulation`）**: React Three Fiber による 3D 歩行シミュレーション。倍速（1 / 2 / 5 / 10 / 50 倍速）切り替え、生存個体数・安定歩行率・最高距離のテレメトリ、スクロールログを表示します。世代交代後の描画が止まらないよう Ref で状態を同期しています。
5. **世代分析画面（`/analysis`）**: 第 1 世代→最終世代の Before/After 比較カードと、上位個体のランキング。
6. **最終診断画面（`/result`）**: S〜C の評価ランク、改善された点・弱点・おすすめ設定、および遺伝的アルゴリズムの仕組みを説明するアコーディオン。

## 技術構成

* **フロントエンド**: React 19, React Router v7
* **3D 描画**: React Three Fiber（`@react-three/fiber`）, drei（`@react-three/drei`）, Three.js
* **物理・進化ロジック**: 自作の簡易歩行物理（サイン波ベースの歩容と重心バランス）と遺伝的アルゴリズム（エリート選抜＋突然変異）を TypeScript で実装
* **ビルドツール / 言語**: Vite 8, TypeScript（Strict Mode）
* **スタイリング**: Vanilla CSS（CSS 変数によるテーマ設計、Google Fonts の Outfit / Inter）

## ローカル起動方法

事前に [Node.js](https://nodejs.org/) をインストールしてください。

1. リポジトリのクローン
   ```bash
   git clone https://github.com/008sunset-gif/evowalker-ai.git
   cd evowalker-ai
   ```
2. 依存関係のインストール
   ```bash
   npm install
   ```
3. 開発サーバーの起動
   ```bash
   npm run dev
   ```
   起動後、ブラウザで `http://localhost:5173/` を開きます。
4. 本番ビルド
   ```bash
   npm run build
   ```
   `dist/` ディレクトリに静的ファイルが出力されます。
