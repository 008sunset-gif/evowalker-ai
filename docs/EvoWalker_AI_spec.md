# EvoWalker AI 技術仕様書

> 本書は実装意図の整理および公開準備のための技術仕様まとめです。記載内容はすべて実装に準拠しており、未使用ライブラリや未実装の手法は記載していません。

---

## 1. アプリ概要

**EvoWalker AI** は、**遺伝的アルゴリズム（GA）**によって二足歩行ロボットの個体群が、世代交代を繰り返しながら「まっすぐ安定して歩く」技術を学習・進化させていく過程を、3Dビジュアルと数理データで観察・分析できる **Webアプリケーション** です。

- 第1世代はランダムな歩き方で転倒・コースアウト・障害物衝突を繰り返す
- 成績の良い個体の遺伝子が次世代へ引き継がれ、世代ごとに歩行が改善していく
- その「試行錯誤と進化」の過程をリアルタイム3D描画とグラフで可視化する

インストール不要・フロントエンド完結（バックエンド/DBなし）で動作します。

---

## 2. 開発目的

- **AI学習プロセスの直感的な可視化**：遺伝的アルゴリズムは結果だけを見ても「なぜその挙動になったか」が分かりにくい。世代ごとの前進距離・ゴール率・転倒率などの変化をリアルタイムに見せ、進化を体感できる教育デモに落とし込む。
- **フロントエンド/3D/アルゴリズム実装の技術検証**：React と React Three Fiber によるリアルタイム3D描画、外部物理エンジンに頼らない自作の簡易歩行物理、Pure TypeScript の遺伝的アルゴリズムを一貫して実装する。
- **個人開発として制作**：Web開発・プログラミング・UI/UX設計の工夫を、公開作品として整理・提示する。

## 3. 使用技術（技術スタック）

| 区分 | 採用技術 |
| :--- | :--- |
| フロントエンド | React 19.x、React Router v7.x |
| 3D描画 | **React Three Fiber**（`@react-three/fiber`）、drei（`@react-three/drei`）、**Three.js** |
| 物理・進化ロジック | **自作の簡易歩行物理**（サイン波ベースの歩容＋重心バランス）＋**遺伝的アルゴリズム** を Pure TypeScript で実装 |
| ビルド／言語 | Vite 8.x、TypeScript（Strict Mode） |
| スタイリング | Vanilla CSS（CSS変数によるライトテーマ、Google Fonts: Outfit / Inter / Noto Sans JP） |

> 補足（正確性のための明記）：**ニューラルネットワーク・強化学習は使用していません。** 物理は **Rapier 等の外部物理エンジンを使わず自作**しています。障害物回避も「センサーで検知して避ける」専用ロジックではなく、後述の**歩行パターン遺伝子**の差で軌道が変わる設計です。GAに**交叉（Crossover）は実装しておらず**、上位個体の保護＋突然変異＋少数のランダム新規個体で進化させています。

## 4. 画面構成（7画面）

React Router による7画面構成。未設定で直接URLアクセスした場合は前段画面へリダイレクトするガードを実装。

1. **トップ（`/`）**：アプリ概要とスタートCTA。歩行ロボットの3Dプレビュー。
2. **レーン選択（`/scenario`）**：3つの環境を選択（内部 scenarioId → 表示名）。
   - `city` → **直線レーン**（広い／初級）
   - `highway` → **細道レーン**（狭い／中級。レーン外へ逸れると脱落しやすい）
   - `circuit` → **障害物レーン**（立体ブロック障害物あり／上級）
3. **AI設定（`/config`）**：歩行方針（安定重視／バランス／スピード重視）、学習スタイル（変異の強さ＝慎重／標準／積極的）、同時歩行個体数（15/30/50）、最大学習世代数（10/30/50）。
4. **ブリーフィング（`/briefing`）**：選択条件と初期パラメータのバイアス確認。
5. **歩行シミュレーション（`/simulation`）**：リアルタイム3D。左上HUD・倍速再生（1/2/5/10/50倍速）・追従カメラ・システムログ。
6. **進化分析（`/analysis`）**：世代推移のSVGグラフと初期 vs 最終の比較、上位個体ランキング。
7. **最終診断（`/result`）**：ルールベース評価（S〜C）と改善点・弱点・次回アドバイス、GAの仕組み解説。

## 5. 遺伝的アルゴリズムの仕組み

各個体（ロボット）の「歩き方」を数値パラメータ群＝**ゲノム（DNA）**として表現し、適合度（スコア）の高い個体の遺伝子を次世代へ引き継ぐことで集団全体の歩行を改善します。

- **適合度評価（Fitness）**：前進距離・生存時間・レーン中央維持・姿勢安定をプラス評価、転倒/障害物衝突/コースアウト/立ち往生をマイナス評価、ゴール到達をボーナス評価。
- **エリート選抜（Elitism）**：前世代の最高成績個体のゲノムを**無変更で1体だけ**次世代へ完全コピー（エリート保護）。
- **親個体の選抜**：**上位個体（おおよそ上位20%、最小3〜最大6体）**からランダムに親を選ぶ。1体の優秀個体に集団が偏りすぎないようにする。
- **突然変異（Mutation）**：親ゲノムに、学習スタイル（慎重/標準/積極的 = `5% / 15% / 30%`）に応じた正規分布風ノイズを加え、各値は上下限でクランプ。
- **ランダム新規個体**：毎世代、末尾の**約10%**を完全ランダムな新規個体に置き換え、多様性（探索）を維持。
- **交叉（Crossover）は不使用**：挙動の説明性と実装の安定性を優先し、「上位個体の保護＋突然変異＋少数のランダム新規個体」というシンプルなGA構成を採用。

---

## 6. ゲノムパラメータ（12個）

各個体の歩行特性を以下の12パラメータで定義（初期世代は広く分散させ、突然変異・選抜で改善）。

| パラメータ | 意味 |
| :--- | :--- |
| `strideLength` | 歩幅 |
| `stepRhythm` | 歩行リズム（足の周期） |
| `balanceCorrection` | 姿勢補正力（傾きを中央へ戻す力） |
| `lateralDrift` | 左右へ流れる癖 |
| `wobbleStrength` | ふらつき量（生まれつきのノイズ） |
| `speedFactor` | 前進速度のベース（最低限の前進力を確保） |
| `steeringBias` | 左寄り／右寄りの偏り |
| `recoveryAbility` | バランスを崩した時の立て直す力 |
| `lateralAmplitude` | 左右に蛇行する大きさ |
| `lateralFrequency` | 蛇行の周期 |
| `lateralPhase` | 蛇行の初期位相（揺れ始めの個体差） |
| `centerPull` | レーン中央へ戻ろうとする力 |

**障害物回避の考え方**：障害物を「検知して避ける」専用ロジックは持たず、上記の歩行パターン遺伝子（蛇行・流れ・中央復帰など）の違いによって各個体の軌道が変わり、**結果的に**障害物を避けられた個体・ぶつかる個体・コースアウトする個体が自然に生まれます。避けやすい歩き方の遺伝子が世代交代で選抜・蓄積されます。

---

## 7. 世代交代ロジック

1. 1世代の全個体をシミュレーション（各個体が停止＝失敗/ゴール、または時間切れで世代終了）。
2. 全個体の `finalScore` を算出しソート。
3. 上記GAルール（エリート保護1体＋上位3〜6体からの突然変異＋約10%の新規個体）で次世代ゲノムを生成。
4. 次世代を初期化して再スタート。
5. **`currentGeneration` が `maxGenerations` に到達したら終了**（最終診断へ）。世代カウンタは権威的なRefでクランプ管理し、完了フラグで遅延処理による超過生成を防止（例：最大10世代設定で「11/10」「16/10」にならない）。

## 8. 物理判定（自作の簡易歩行物理）

外部物理エンジンを使わず、毎フレーム（倍速時はサブステップごと）に以下を計算します。

- **歩容**：各個体の歩行リズム（`stepRhythm`）に沿ったサイン波で脚の動き・推進力・ふらつきを生成。`speedFactor` で前進。
- **重心バランス**：重心の傾き `wobbleAngle` を、`balanceCorrection`・`recoveryAbility` が中央へ戻そうとする。
- **横移動**：`lateralAmplitude/Frequency/Phase` による蛇行＋`lateralDrift/steeringBias` の流れ＋`centerPull` の中央復帰の合成で横位置 `y` が決まる。
- **座標統一**：物理の前進方向 `x`／横ずれ `y` と、3D描画座標を `worldZ = -walker.y` の一貫した変換で結び付け、当たり判定と見た目を一致させる。
- **障害物判定**：障害物レーンのみ。障害物データ `OBSTACLES_3D` を**描画と物理判定で同一参照**し、矩形（AABB）＋ロボット半径で衝突を判定（見た目と判定が一致＝すり抜けなし）。
- **倍速対応**：1〜50倍速は「1フレームを runSpeed 回のサブステップに分割し、各サブステップで1tickずつ判定」する方式。位置を一括で飛ばさないため、衝突・コースアウト判定が抜け落ちない。

## 9. 失敗・成功の定義（HUDで別カウント）

各個体は単一の `status` を持ち、原因が排他的にカウントされます（二重計上なし）。

| 状態 | 定義 | 失敗の意味 |
| :--- | :--- | :--- |
| **ゴール（goal）** | 前進距離がゴール地点に到達 | 完走（成功） |
| **転倒（fallen）** | 重心の傾き `|wobbleAngle|` が限界角を超過 | 姿勢制御の失敗 |
| **障害物衝突（obstacleHit）** | 障害物の矩形範囲に接触（障害物レーン） | 回避（軌道選択）の失敗 |
| **コースアウト（outOfLane）** | 横位置がレーン半幅を超過 | 直進性・復帰力の失敗 |
| **立ち往生（stalled）** | 一定時間ほとんど前進できない | 推進力・歩行リズムの失敗 |

見た目でも区別：転倒＝横倒し（赤系）／衝突＝直立のまま停止＋のけぞり＋赤い警告リング／コースアウト＝レーン外へ沈下。

**スコア式（概略）**：`前進距離×2.5 ＋ 生存時間×0.1 ＋ 中央維持ボーナス×0.5 ＋ 姿勢安定ボーナス ＋ 障害物通過ボーナス(通過数×60) − ペナルティ`。ペナルティは 転倒150／障害物衝突250／コースアウト200／立ち往生100、ゴールは加点（−350）。距離は内部座標（0〜300）を `toMeters` でメートル換算（0〜60m）に統一。

## 10. 3D表現

- `@react-three/fiber` / `@react-three/drei` で、歩行ロボット個体群・レーン環境（路面・境界フェンス・ゴールゲート・立体障害物）・追従カメラ（`DynamicCamera`）・ライティング（`SceneSetup`）を構築。
- ロボットは頭部・胴体・腕・脚の多関節を歩行リズムに合わせて動かし、傾き（ふらつき）が動的に変化。
- パーティクル演出（`ParticleSystem3D`）：着地時の波紋・煙、転倒時の火花、ゴール時の紙吹雪。高速再生時は装飾パーティクルのみ抑制（物理判定は維持）。
- `useFrame`（3D描画ループ）と React の状態更新が競合してフリーズしないよう、`isRunningRef` 等の Ref 同期と `isTransitioningRef` セマフォで安全に同期。

## 11. Analysis / Result の表示内容

すべて実シミュレーション結果（`GenerationResult` 配列）に基づく**実データ**表示です（ダミーデータではありません）。

**Analysis（進化分析）**
- 結論ヘッドライン：第1世代→最終世代の主要数値の変化と判定（改善あり／停滞／悪化）。
- サマリーカード5項目：平均前進距離・ゴール率・転倒率・コースアウト率・集団改善度。
- 主要グラフ（2×2 SVG折れ線）：平均前進距離・ゴール率・転倒率・コースアウト率の世代推移。
- 安定歩行率グラフ（既定展開の折りたたみ）。
- 上位個体ランキング、初期世代 vs 最終世代の比較表。

**Result（最終診断）**
- ルールベース評価エンジンが全世代結果を解析し、総合レーティング（S〜C）と歩行タイプを判定。
- 「改善された点」「まだ残る弱点」「次回おすすめ設定」を結論ファーストで提示。
- 遺伝的アルゴリズムの仕組み・UI/UX設計意図をアコーディオンで解説。

## 12. 技術的なポイント（実装意図の整理）

- **React Three Fiber によるリアルタイム3D描画**：状態管理と描画ループ（`useFrame`）の同期、Ref同期による stale closure 回避、世代交代時のフリーズ防止セマフォ。
- **自作の簡易歩行物理**：外部物理エンジンを使わず、サイン波歩容＋重心バランスでロボットの歩行を表現。物理座標と3D描画座標を `worldZ = -y` で統一し、当たり判定と見た目を一致させた。
- **遺伝的アルゴリズムの設計判断**：交叉を使わず「上位保護＋突然変異＋ランダム新規個体」を採用した理由（挙動の説明性・実装の安定性・局所最適回避）。障害物回避を専用AIにせず、歩行パターン遺伝子の差で軌道が変わる“創発的”な設計にした意図。
- **失敗原因の分離設計**：転倒／障害物衝突／コースアウト／立ち往生を排他ステータスで管理し、二重計上なく可視化（学習で何が改善したかを説明可能にする）。
- **性能と正確性の両立**：1〜50倍速をサブステップ実行で実現し、当たり判定を飛ばさない設計。
- **UI/UX設計**：7画面の導線、結論ファーストの分析・診断、ライトテーマ、待機ストレスを下げる倍速・途中分析導線。

### 設計判断に関する補足（想定される技術的な論点）
- 「交叉を使わない理由は？」→ 多様性は突然変異＋毎世代のランダム新規個体で確保。実装と挙動説明をシンプルに保つ意図。
- 「本当に学習している？」→ 適合度に基づく選抜で、世代ごとに平均前進距離・ゴール率が上昇し転倒率が低下する実データを提示。
- 「すり抜けは起きない？」→ 描画と物理が同一障害物データ（AABB）を参照し、倍速でもサブステップ判定するため発生しない。
