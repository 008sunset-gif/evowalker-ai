## P1-1: 歩容に "上手さ" を反映（2026-06-11）

### 背景 / 問題
脚アニメが `stepRhythm` と `strideLength` の2遺伝子しか使っておらず、
"上手さ" の遺伝子（`balanceCorrection` / `recoveryAbility` / `wobbleStrength`）が
手足の動きに出ていなかった。そのため第1世代のヘタな個体も最終世代の洗練個体も
**同じ綺麗なサイン波の行進**に見え、「学習している／進化している」体感がなかった。

### 変更方針
- **GAロジック・物理計算（`walkerPhysics3D.ts` / `evolution.ts` / `vehiclePhysics.ts`）には一切触れない。**
- **描画レイヤー（`src/components/three/WalkerRobot3D.tsx`）のマッピングのみ拡張。**

### 実装内容（`WalkerRobot3D.tsx` の `useFrame` 内・歩行アニメ分岐）
1. "上手さ" 遺伝子を正規化し、歩きの「ガクつき量」`instability`(0〜1)を合成：
   ```
   balance01   = clamp(balanceCorrection / 1.5)
   recovery01  = clamp(recoveryAbility   / 1.2)
   wobble01    = clamp(wobbleStrength    / 2.5)
   instability = wobble01*0.5 + (1-balance01)*0.3 + (1-recovery01)*0.2   // 0=達人, 1=新人
   ```
2. `instability` を以下の見た目に反映（高いほど不安定に見える）：
   - **上体の横揺れ(swayZ)**：高周波の小刻みな揺れ + `recovery` 低で低周波の戻り遅れ（カクつき）
   - **上体の前後ぐらつき(pitchX)**：group.rotation.x に付与
   - **上下バウンド(roughBob)**：弾みが乱れる
   - **歩幅の左右非対称ジッター**：thigh角に乗算（不安定なほど左右バラバラ）
   - **腕のバタつき(armFlail)**：腕スイングに加算
3. 揺れは `Math.random()` を使わず、**非整数倍音サインの重ね合わせ + 個体ごとの位相オフセット
   (`walker.id`)** で生成（フレーム間フリッカを避け、群れの単調さも防ぐ）。
4. 物理由来のチルト(`-walker.wobbleAngle`)・向き(`walker.angle`)・接地・転倒/ゴール等の
   既存分岐はそのまま維持。

### 検証
- `npm run build`（`tsc -b && vite build`）が成功。型エラーなし。既存ロジック不変。
- マッピングの決定論的検証（ヘタ個体 vs 洗練個体のピーク揺れ量）：

  | 指標 | 第1世代(ヘタ) | 最終世代(上手) |
  |---|---|---|
  | instability | 0.90 | 0.08 |
  | 上体横揺れ | 25.0° | 2.2° |
  | 前後ぐらつき | 8.8° | 0.7° |
  | 上下バウンド | 0.41 | 0.03 |
  | 腕バタつき | 18.1° | 1.5° |
  | 歩幅ばらつき | ±25% | ±2% |

  → ヘタ個体は大きくぐらつき腕もバタつくのに対し、洗練個体はほぼ元の綺麗な行進に収束。
  世代が進み選抜が効くほど `instability` が下がり、歩きが洗練されて見える。

### 影響範囲 / リスク
- 描画のみ。シミュレーション結果・スコア・分析数値・GAの収束には影響しない。
- TopPage のヒーローデモ個体（balance/recovery=1.0, wobble=0.2）は instability≈0.17 で
  ごく軽い "生きている" 揺れが付く程度。
- 残課題：3D実機での視覚確認（第1世代 vs 最終世代の録画）は未実施。dev起動での目視確認が望ましい。

### 未着手（次回候補）
P0-1 ゴースト&軌跡 / P0-2 進化曲線3本線 / P0-3 前後比較リプレイ / P0-4 結果トーン /
P1-2 転倒の "ぐらつき→復帰"（※本体に踏み込む） など。

## トップページ（ヒーロー）のプロ品質リデザイン（2026-06-11）

### 背景 / 問題
- ロゴが全大文字「EVOWALKER AI」(Orbitron) でゲーム的・習作感。
- ヘッダーの英字バッジ「Evolution Simulator」が冗長。
- ヒーロー背景にAI生成ストック画像 `hero_main.png` を使用 → 量産AI感の主因。
- 大量のインラインスタイルで保守性・一貫性が低い。レスポンシブ未対応・a11y未対応。

### 変更方針
- **中身（GA・3D・物理）には触らない。** 見た目・テキスト・ボタンのみ。
- インラインスタイルを廃し、`index.css` にクラスベースで再設計。

### 実装内容
**`components/Layout.tsx`（ヘッダー）**
- ロゴを全大文字 → ミックスケース **「EvoWalker AI」**、フォントを Orbitron → 見出しフォント(M PLUS 1p)に変更し letter-spacing を詰めて上品に。
- 英字バッジは英語のまま維持しつつ、モバイルでは非表示（CSS）。

**`pages/TopPage.tsx`（ヒーロー）**
- AIストック背景画像を**完全に廃止**。
- マークアップをクラスベース（`.hero` / `.hero__inner` / `.hero__copy` / `.hero__visual` 等）に全面刷新。
- コピーを自然な日本語に：見出し「転んで、選ばれて、次の世代へ。歩行AIの進化を、観察する。」、
  リード文を簡潔化、3ステップを「試す／選ばれる／進化する」チップに整理。
- CTAを「シミュレーションを始める →」のグラデーションボタン＋補足「登録不要・ブラウザですぐ動きます」に。
- 中二病的な英字バッジ "EVOLUTIONARY AI SIMULATOR" を上品なアイブロウ "遺伝的アルゴリズム × 3D シミュレーション" に。
- ヒーローの3Dロボットを `isElite={true}` → `false` に変更し、金色 → **ブランドのシアン**に統一（3D/GAロジックは不変、表示フラグのみ）。3Dシーン(`<Canvas>`)の中身は完全に維持。

**`index.css`**
- TOP PAGE セクションを近代的なヒーロー用スタイルに全面書き換え（他ページ共用の `.portfolio-badge` は保持）。
  グラデーションCTA、プロダクトショット風ステージ（柔らかいグロー＋影）、アイブロウ等。
- **レスポンシブ**：`@media (max-width:900px)` で2カラム→1カラム縦積み＆ビジュアル先頭化、
  `@media (max-width:520px)` でステップ縦積み・CTA全幅・見出し縮小。`@media (max-width:640px)` でモバイル左右余白を 48px→16px。
- **a11y**：`@media (prefers-reduced-motion: reduce)` を追加し fade-in / pulse-glow 等のアニメを抑制。
- **バグ修正**：Grid アイテム `.hero__copy` に `min-width: 0` を付与。`min-width:auto` 既定でテキスト幅までトラックが広がり溢れる典型不具合を解消（モバイルで見出しが折り返さず切れていた根因）。

### 検証
- `npm run build`（`tsc -b && vite build`）成功・型エラーなし。既存機能・GA・3Dロジック不変。
- `npm run dev` 起動 → ヘッドレスChromeでデスクトップ(1440)・モバイル幅でスクショ撮影し、2巡以上の自己批評と修正を実施：
  - 1巡目：robotが金色でブランド不一致 → シアンへ。
  - 2巡目：モバイルで見出しが右端で見切れ → 原因をGridの `min-width:auto` と特定し `min-width:0` で修正。
  - 検証幅500pxでデバッグ枠を当て、見出しが完全に折り返し・ステップ縦積み・見切れゼロを確認。
  - 注：ヘッドレスChromeは最小ウィンドウ幅の制約で実390pxを正確に再現できず（画像に広いレイアウトの左側が写る）、
    520pxブレークポイントが効く幅で折り返しを検証。実機390pxはフォント下限(23px)でさらに余裕。
- 結果：AIストック感・全大文字ロゴ・直訳調が解消され、最新SaaS風のヒーローに到達。

## 全面改善 第1弾：デザイン土台（トークン化＋ダーク基調）（2026-06-11）

### 目的
色・余白・影が全インラインのバラバラな値で書かれ、デザインの単一情報源が無かった。
統一トークンに集約し、基調を「洗練されたダーク（Linear / Vercel / Framer系）」に
決め打ちして全画面のちぐはぐを根本解消する。GA・3D・物理ロジックには一切触れない。

### 実装内容
**`index.css` :root — デザイントークン整備**
- 色：アクセントを sky 1系統に集約（`--color-primary:#38bdf8` / `--color-primary-strong:#0ea5e9`）。
  旧 teal(`#4fd1c5`/`rgba(79,209,197)`) の残骸を全排除。violet は補助（stepper等）に限定。
  **未定義で壊れていた `--color-primary-glow` を定義**。
- 背景の階層：`--bg-base/--bg-main/--bg-card/--bg-card-hover/--bg-inset/--bg-elevated`（ダーク）。
- ボーダー：`--border-color`(translucent) / `--border-strong`。
- テキスト：`--text-title/--text-main/--text-muted/--text-faint`（ダーク用に反転）。
- スケール追加：`--space-1〜8` / `--radius-sm〜2xl,pill` / `--shadow-sm/md/lg/glow` /
  `--fs-*`(サイズ) / `--fw-*`(ウェイト)。

**ダーク基調への移行**
- `body` を `--bg-base` ＋ sky/violet の微グラデ(radial glow)に。平坦さを避け奥行きを付与。
- グローバル `h1〜h6` の「Orbitron・全大文字・シアン・glow」強制を撤去し、
  見出しフォント＋`--text-title`＋字間調整の素直な指定に変更（各ページのインライン打ち消し問題を解消）。
- 共有CSS（`.card`/`.btn-primary`/`.btn-secondary`/stepper/`.scenario-card`/`.option-card`/
  `.segmented-control`/`.difficulty-badge`/hero各種）をトークン参照のダークに統一。
  teal・真っ黒な影(`rgba(0,0,0,0.5)`)などの残骸を一掃。

**各ページのインライン色をトークン参照に変換**
- ScenarioPage / ConfigPage / BriefingPage / AnalysisPage / ResultPage / TopPage(hero) の
  ハードコード色（`#0f172a`/`#ffffff`/`#f8fafc`/`#f1f5f9`/`#e2e8f0`/`#64748b`/`#475569`/`#334155` 等）を
  セマンティックトークンへ置換。
- 多義的な値は文脈別に対応：暗パネル背景(`#0f172a`/`#ffffff`)→`--bg-card`/`--bg-inset`、
  Briefingの brain card は `--bg-elevated→--bg-inset` グラデに。
- **AnalysisPageのSVGチャート**：`fill="..."`/`stroke="..."` はプレゼンテーション属性で CSS変数が効かないため、
  literal な明色（`#e8edf5`/`#94a3b8`/`rgba(148,163,184,.18)`）に置換し可読性を確保。
- SimulationPage の HUD は既にダークのため非干渉（白文字/muted はダーク上で適正）。

### 検証（スクショ＋3巡 自己批評）
- `npm run build` 成功・型エラーなし。GA/3D/物理ロジックは無変更。
- CDP(Chrome DevTools Protocol)で実フローをクリック操作し、全6画面を撮影

- 巡1：ヘッダー(ダーク)とコンテンツ(白)のトーン分裂が解消され、全画面が一貫したダークに。最大目標を達成。
- 巡2：invisible text・コントラスト破綻なし。白文字は暗カード上で可読、アクセントは明色で映える。
  teal完全排除、sky単色＋violet補助で統一。
- 巡3（残課題＝次弾）：Config/分析/結果の AI生成ストック画像ヘッダー、CTA階層(戻る/進むが同格)、

## ① 障害物コースの進化（センサー回避）＋ ② 歩行アニメの自然化（2026-06-12）

### ① 障害物を「避けて前進」できる進化（A案：センサー＋遺伝子）— 課題と解決
**課題**：従来の circuit は前方を検知して避ける手段が無く、固定の蛇行がたまたま障害物を外すかの
運任せ。進化しても回避が上達せず完走0に陥りやすかった。中央ブロック（直進だと必ず衝突）と
中央維持ボーナスの選抜圧も噛み合っていなかった。

**解決（GA本体に踏み込む実装）**：
1. 遺伝子に `obstacleAvoidance`（回避の強さ）を1つ追加（`types`/`vehiclePhysics`初期生成/`evolution`変異）。
2. `walkerPhysics3D` に**前方センサー回避**を実装：前方 LOOKAHEAD(60) 以内の障害物の占有帯（±余裕）を集め、
   現在進路が塞がれている時だけ、**レーン内で現在地に最も近い空き帯**へ `avoidVel` で横移動。
   強さは `obstacleAvoidance` に比例。中央ブロックでは左右どちらかの空き、ゲートでは中央の空きへ
   自然に寄る（「最も近い空き帯」を選ぶだけで両方に対応）。
3. 衝突は**従来どおり致命**（避けるべき性質を維持）。回避は横移動のみで前進速度は落とさない＝
   「避けすぎて止まる」も回避。fitnessは到達距離が主で、避けて遠くまで行く個体ほど選抜される。

**検証（ヘッドレスで実モジュールを15世代回した実測）**：
| 世代 | 完走 | 平均到達(m) | 回避遺伝子(平均) | 衝突死 |
|---|---|---|---|---|
| 1 | 1 | 10 | 0.59 | 4 |
| 3 | 3 | 29 | 0.70 | 17 |
| 5 | 11 | 39 | 1.32 | 11 |
| 6 | 14 | 40 | 1.25 | 9 |
| 9 | 14 | 47 | 1.15 | 9 |
- 完走 1→14体、平均到達 10→47m、**回避遺伝子 0.59→1.2+ に上昇（＝選抜で回避が進化）**。
- 衝突死は中盤に増えてから減少：「障害物に届く前に死ぬ→届いてぶつかる→避けを学習」という自然な学習曲線。
- city/highway も正常に進化（完走増加・エラー無し・衝突死0＝障害物が無いため）。GA全体は破綻せず。
- 中盤以降は完走9〜14でやや揺れる（毎世代10%の新規個体注入＋変異ノイズ＋集団収束による）。
  進化の主トレンド（1→6世代）は明確。

### ② 歩行アニメの自然化（描画のみ・GA/物理不変）
`WalkerRobot3D` の歩容を、滑らかな関数のみ（乱数フリッカ不使用）でリアルに：
- 膝：遊脚（踏み出し）で `max(0,sin)²` により大きく屈曲して足を持ち上げ、立脚ではほぼ伸ばす（メリハリ）。
- 足首：踏み出しでつま先上げ・蹴り出しで下げ＋脛の傾きに対し足裏を地面寄りへ補正（ロール）。
- ボビング：両脚接地で沈み立脚中央で持ち上がる `(1-cos(2φ))` の滑らかな上下動。
- 体重移動の左右揺れ（歩調同期 sin）＋上体のひねり（肩腰の逆回転 yaw）＋わずかな前傾。
- **P1-1の「上手さ遺伝子→ガクつき」反映は維持**（instability による swayZ/pitchX/roughBob/歩幅非対称/腕バタつきを上乗せ）。
  ベース歩容が自然になった上で、下手な個体は乱れ・上手い個体は滑らか、の差は保たれる。

### 確認
- `npm run build` 成功・型エラーなし（新遺伝子フィールドの欠落も無し）。
- circuit/city/highway をヘッドレス実走で確認（上表）。3Dシーンの描画も正常（gait_circuit_*）。
