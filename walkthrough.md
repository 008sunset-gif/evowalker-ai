# Walkthrough - Evodrive AI (開発実績と実装詳細)

本ドキュメントは、**Evodrive AI** の開発プロセスにおける主要な設計、実装された技術要素、および現時点（Step 7.6完了時）での動作実績を整理した成果物説明用のウォークスルーです。

---

## 1. 開発実績と実装済み機能

本プロジェクトは、React 19 と TypeScript、HTML5 Canvas API をフルに活用し、完全フロントエンド完結型の自動運転AI進化シミュレーターとして構築されています。

### ① 画面構成と React Router 遷移
*   **7つの基本画面**が構築され、コンテキストの状態に応じたスムーズな遷移を実現しています：
    1.  **トップ画面 (`/`)**：イントロダクションとスタート導線。
    2.  **シナリオ選択画面 (`/scenario`)**：走行コースの選択。
    3.  **AI設定画面 (`/config`)**：性格や走行台数、世代数などのカスタマイズ。
    4.  **ブリーフィング画面 (`/briefing`)**：遺伝的初期バイアスの可視化と確認。
    5.  **シミュレーション画面 (`/simulation`)**：リアルタイム2D自律走行。
    6.  **世代分析画面 (`/analysis`)**：走行データのSVG推移グラフとランキング。
    7.  **最終診断画面 (`/result`)**：ルールベースの運転診断。
*   **ダイレクトURLアクセスガード**：ユーザーが設定を通らずに `/simulation` や `/result` に直接アクセスした場合、強制的に適切な前画面へリダイレクトされる安全設計が実装されています。

### ② HTML5 Canvas 2D 物理演算エンジン
*   **リアルタイム走行物理**：各車両は座標、進行角度（ラジアン）、速度、ステアリング舵角を持ち、カーブでの旋回Gや、目標ポイントまでの距離に応じた加速・減速の物理挙動を毎フレーム計算して滑らかに描画。
*   **シナリオ別環境の構築**：
    *   **市街地**：直線的なグリッドと建物障害物を配したテストコース。
    *   **高速道路**：車線変更を模擬するための対向車・分離帯障害物と、直線的なコース。
    *   **サーキット**：急なヘアピンカーブや連続S字を持つ周回コース。
*   **衝突/クラッシュ判定**：
    *   Canvas境界外への逸脱、障害物矩形への接触、道路（アスファルト外壁）からのコースアウトを毎フレーム線分交点・矩形判定。
    *   クラッシュした車両は速度ゼロで停止し、車体をダークグレーに切り替え、衝突位置に「赤い × 印」を描画。

### ③ 5方向センサーレーキャスト
*   主要車両（AI-01）に、進行角度を基準とした5つの角度（-45度、-22.5度、0度、22.5度、45度）の測距センサーを接続。
*   各センサーラインの先端から前方の道路境界または障害物までの距離をミリ秒単位でレイキャスト（線分交点）スキャン。
*   障害物や境界までの距離が近づくにつれて、センサー線の描画色が **シアン（安全） ➔ イエロー（注意） ➔ レッド（衝突危険）** に動的に変化する可視化表示。

### ④ 遺伝的アルゴリズムと世代交代ループ
*   **定量的スコア評価 (`finalScore`)**：
    *   `スコア ＝ 通過WP数 × 1000 ＋ 走行距離 × 0.5 ＋ 生存時間 × 0.2 － クラッシュペナルティ(500)`
    *   単に長く走るだけでなく、コースに沿って前進すること（Waypoint通過数）に最大の重みを与えることで、適切な進化圧を発生。
*   **エリート選抜（Elitism）**：
    *   前世代の全車両をスコア順にソートし、上位20%を次世代の親として選抜。さらに、前世代で最高得点を出したベスト個体のゲノム特性は次世代へ完全コピー（エリート保護）され、進化の退行を防ぎます。
*   **突然変異（Mutation）**：
    *   エリート以外の個体は、親ゲノムを基に学習スピード（ゆっくり、標準、高速）に対応した突然変異率（`5% / 15% / 30%`）で、正規分布風のランダムノイズ（$-1.5$ 〜 $+1.5$）を加算し、多様な特性を持つ次世代パラメータを生成。
*   **React StrictMode二重実行防止ガード**：
    *   アニメーションループ内で「全員停止」「時間切れ（1800F）」「ゴール到達」の終了判定を監視する際、StrictModeによる多重実行で同じ世代データが重複保存されたり、世代カウンターが不自然に飛ぶ現象を、コンポーネント内 Ref の `isTransitioningRef.current`（セマフォ）を用いて厳密にブロック。

### ⑤ SVG推移グラフと動的診断
*   **データ可視化**：世代ごとの「ベストスコア（シアン・実線）」「平均スコア（パープル・破線）」「クラッシュ台数（レッド・実線）」の推移を、1つのSVG要素内に重ねて折れ線グラフとして描画。初期世代からのスコア改善率や最高到達WP数の変化を数値で併記。
*   **ルールベース診断**：最終世代完了時のスコア改善率、クラッシュ率の変化を基に、AIモデルの特性を4タイプに動的判定して出力（総合運転レーティングS/A/B/C、走行安定性向上フレーズ、残る課題やアドバイスのレポート生成）。

## 2. 進化挙動のチューニング実績 (Step 7.6)

初期世代での「未熟でバラつきのある走り」から、後半世代での「滑らかで安定した完走・周回」への変化を劇的に見せるために、物理挙動とゲノムパラメータの相関を精密に調整しました。

1.  **多様性ノイズの拡大（±35%）**
    *   第1世代では、スピードを出しすぎる車両や、ステアリングのブレが極端な車両が数多く生まれ、スタート直後の急カーブでクラッシュが多発します。
2.  **ゲノムの淘汰とブレの収束**
    *   ブレ（`noiseFactor`）が小さく、カーブ手前での減速（`brakeSensitivity`）が適切で、適度なハンドリング反応（`steeringResponsiveness`）を持つ車両が自然と生き残り、世代が進むにつれて走行ブレが綺麗に抑えられた「超安定モデル」へ収束します。
3.  **劇的な改善率の可視化**
    *   初期世代のベストスコアが低く抑えられ、最終世代では完走車が多数出現するため、最高スコアが **+20% 〜 +150% 以上向上** し、生存率が飛躍的にアップする「AIの進化プロセス」が数値・ビジュアル双方で体感できるようになりました。
