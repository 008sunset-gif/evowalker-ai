# Walkthrough - EvoWalker AI (開発実績と実装詳細)

本ドキュメントは、**EvoWalker AI** の開発プロセスにおける主要な設計、実装された技術要素、および現時点での動作実績を整理した成果物説明用のウォークスルーです。

---

## 1. 開発実績と実装済み機能

本プロジェクトは、React 19 と TypeScript、React Three Fiber（Three.js）をフルに活用し、完全フロントエンド完結型の **AI歩行進化シミュレーター** として構築されています。

### ① 画面構成と React Router 遷移
*   **7つの基本画面**が構築され、コンテキストの状態に応じたスムーズな遷移を実現しています：
    1.  **トップ画面 (`/`)**：イントロダクションとスタート導線。
    2.  **レーン選択画面 (`/scenario`)**：歩行レーン環境（直線・細道・障害物）の選択。
    3.  **AI設定画面 (`/config`)**：歩行方針、学習スタイル（変異の強さ）、同時歩行個体数、世代数のカスタマイズ。
    4.  **ブリーフィング画面 (`/briefing`)**：設定から変換された初期パラメータ（遺伝的バイアス）の可視化と確認。
    5.  **シミュレーション画面 (`/simulation`)**：React Three Fiber によるリアルタイム3D歩行。
    6.  **世代分析画面 (`/analysis`)**：歩行データのSVG推移グラフと上位個体ランキング。
    7.  **最終診断画面 (`/result`)**：ルールベースの歩行診断レポート。
*   **ダイレクトURLアクセスガード**：ユーザーが設定を通らずに `/simulation` や `/result` に直接アクセスした場合、強制的に適切な前画面へリダイレクトされる安全設計が実装されています。

### ② 自作の簡易歩行物理エンジン (Pure TypeScript)
*   **歩容（歩行運動）の再現**：各個体は前進方向の座標 `x`、横ずれ `y`、進行角度 `angle`、重心の傾き `wobbleAngle` を持ち、自身の歩行リズム（`stepRhythm`）に沿ったサイン波で脚の動き・推進力・ふらつきを毎フレーム計算します。重心の傾きは姿勢補正力（`balanceCorrection`）とリカバリー力（`recoveryAbility`）によって中央へ戻ろうとします。
*   **レーン別環境の構築**：
    *   **直線レーン**：障害物のない標準的な広いレーン。
    *   **細道レーン**：レーン幅が狭く、ふらつくとすぐに脱落するレーン（レーン半幅を狭く設定）。
    *   **障害物レーン**：レーン上に立体ブロック障害物が点在する難関レーン。
*   **転倒・脱落・衝突判定**：
    *   重心の傾き `|wobbleAngle|` が限界角度を超えると**転倒**。
    *   レーン外側境界を越えると**レーン脱落（コースアウト）**。
    *   障害物の矩形範囲に接触すると**衝突**（障害物レーンのみ）。
    *   一定時間ほとんど前進できない個体は**立ち往生**として停止。
    *   いずれも物理座標と3D描画座標を一貫した変換（`worldZ = -y`）で結び付け、当たり判定と見た目を一致させています。

### ③ React Three Fiber による3D描画と演出
*   **3Dシーン構成**：歩行ロボット個体群、レーン環境（路面・境界・ゴール・障害物）、追従カメラ（`DynamicCamera`）、ライティング（`SceneSetup`）を `@react-three/fiber` / `@react-three/drei` で構築。
*   **パーティクル演出**：足元の着地時の波紋・煙、転倒時の火花、ゴール到達時のカラフルな紙吹雪を `ParticleSystem3D` で描画し、成否を視覚的に強調。
*   **状態同期設計**：3D描画ループ（`useFrame`）と React の状態更新が競合してフリーズしないよう、`isRunningRef` / `runSpeedRef` / `currentGenerationRef` などの Ref を `useEffect` で同期し、ループ内では `.current` を直接参照する方式で stale closure を回避しています。

### ④ 遺伝的アルゴリズムと世代交代ループ
*   **定量的スコア評価 (`finalScore`)**：
    *   `スコア ＝ 前進距離 × 2.5 ＋ 生存時間 × 0.1 ＋ レーン中央維持ボーナス ＋ 姿勢安定ボーナス － ペナルティ（転倒/脱落/立ち往生）＋ ゴールボーナス`
    *   単に長く生き残るだけでなく、まっすぐ前進すること・中央を保つこと・姿勢を安定させることに重みを与え、適切な進化圧を発生させています。
*   **エリート選抜（Elitism）**：
    *   前世代の全個体をスコア順にソートし、上位20%を次世代の親として選抜。さらに、前世代で最高得点を出したベスト個体のゲノムは次世代へ完全コピー（エリート保護）され、進化の退行を防ぎます。
*   **突然変異（Mutation）**：
    *   エリート以外の個体は、親ゲノムを基に学習スタイル（慎重 / 標準 / 積極的）に対応した突然変異率（`5% / 15% / 30%`）で、正規分布風のランダムノイズを加算し、多様な特性を持つ次世代パラメータを生成。各パラメータは下限・上限でクランプして異常値を防ぎます。
    *   ※交叉（Crossover）やニューラルネットワークは用いず、「エリート選抜＋突然変異」によるシンプルな進化モデルです。
*   **世代交代の二重実行防止ガード**：
    *   「全個体停止」または「時間切れ」を検知して世代を確定する際、同じ世代データが重複保存されないよう、コンポーネント内 Ref の `isTransitioningRef.current` をセマフォとして用いて厳密にブロックしています。

### ⑤ SVG推移グラフと動的診断
*   **データ可視化**：世代ごとの「平均前進距離」「ゴール率」「安定歩行率」「転倒率」「コースアウト率」の推移を、自作のSVG折れ線グラフとして描画。初期世代からの改善を数値で併記します。
*   **ルールベース診断**：最終世代完了時のスコア改善率・転倒率の変化を基に、AIモデルの特性を4タイプに動的判定して出力（総合歩行レーティング S/A/B/C、改善点・残課題・次回アドバイスのレポート生成）。

## 2. 進化挙動のチューニング実績

初期世代での「未熟でふらつく歩行」から、後半世代での「滑らかで安定した完走」への変化を見せるために、物理挙動とゲノムパラメータの相関を調整しました。

1.  **多様性ノイズによる初期世代の失敗体験**
    *   第1世代では、歩幅やリズムが極端な個体、ふらつきの大きい個体が多く生まれ、数歩で転倒・脱落が多発します。
2.  **ゲノムの淘汰とふらつきの収束**
    *   ふらつき量（`wobbleStrength`）が小さく、姿勢補正力（`balanceCorrection`）やリカバリー力（`recoveryAbility`）が適切で、直進性の高い個体が自然と生き残り、世代が進むにつれて安定した歩行へ収束します。
3.  **改善プロセスの可視化**
    *   世代を追うごとに完走（ゴール到達）個体が増え、平均前進距離・安定歩行率が上昇し、転倒率が低下する「AIの進化プロセス」を数値・3Dビジュアル双方で体感できるようにしました。
